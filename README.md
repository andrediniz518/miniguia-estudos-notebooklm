## Engenharia de prompt

**Prompt 1 - Revisão da lógica de programação.**

_*Pergunta: Me ajude a revisar o que é lógica de programação.*_

**Resposta da IA:**

Para revisar o que é lógica de programação, pense nela como o conjunto de regras e instruções que define como um programa deve se comportar para resolver um problema

Abaixo, apresento os pilares principais com base nos documentos:

1. **Variáveis e Tipos de Dados** (As "Gavetas" de Informação) A programação começa com a capacidade de guardar informações.
Em Python, uma variável é como um nome que aponta para um objeto na memória do computador.

Tipos comuns: Você pode guardar números inteiros (int), números com vírgula (float), textos (str) ou valores lógicos (bool, que são apenas "Verdadeiro" ou "Falso")


Tipagem Dinâmica: Uma característica importante é que você não precisa dizer ao computador qual o tipo da "gaveta" antes de usar; o sistema identifica isso automaticamente quando você atribui um valor


2. **Estruturas Condicionais** (Tomada de Decisão)
A lógica permite que o programa "decida" qual caminho seguir através de comandos como if (se), elif (senão se) e else (senão)

Como funciona: O programa testa uma condição usando operadores (como == para igual, > para maior ou != para diferente)

Exemplo: Se uma condição for verdadeira, o programa executa um bloco de código; se for falsa, ele pode seguir um caminho alternativo definido pelo else

3. **Estruturas de Repetição** (Automação de Tarefas)
Frequentemente, a lógica exige que uma mesma ação seja repetida várias vezes até que um objetivo seja alcançado
. Para isso, usamos os loops
while: Executa um conjunto de instruções enquanto uma condição for verdadeira
for: É usado para percorrer itens de uma coleção (como uma lista de nomes) e executar uma ação para cada um deles

4. **Funções (Organização e Reuso)**
Para não precisar reescrever a mesma lógica várias vezes, usamos as funções
. Elas são blocos de código organizados e reutilizáveis que realizam uma ação específica
. Você pode imaginar uma função como um mecanismo que recebe uma entrada (input), processa essa informação e devolve uma saída (output)

Em resumo, a lógica de programação consiste em organizar essas peças (variáveis, decisões, repetições e funções) em uma sequência que o computador consiga seguir para atingir um resultado esperado

## Variação do Prompt 1

- Explique o que é lógica de programação para um iniciante que nunca programou antes, utilizando exemplos
do cotidiano em python para melhor entendimento. Depois me passa um exercicio simples para ver se eu entendi.

**Reposta da IA:**

A lógica de programação é o conjunto de regras e instruções que você define para que um computador consiga resolver um problema ou realizar uma tarefa específica

. Pense nela como uma receita de bolo ou um manual de instruções: se os passos não estiverem na ordem correta ou forem confusos, o resultado final não será o esperado.

Para entender como isso funciona no Python, vamos usar analogias do dia a dia:

1. **Variáveis: As "Etiquetas" de Informação**
Imagine que você tem várias caixas em sua casa. Para saber o que tem dentro de cada uma, você coloca uma etiqueta. Na programação, as variáveis são nomes que apontam para valores guardados na memória do computador


No cotidiano: Você guarda o nome de um amigo em um contato do celular. O nome do contato é a variável, e o número de telefone é o dado.
Em Python: nome = "Alice" ou idade = 25

2. **Estruturas Condicionais: Tomando Decisões**
A lógica permite que o programa "decida" o que fazer com base em uma condição, usando os comandos if (se) e else (senão)

No cotidiano: Se o sinal estiver verde, eu atravesso a rua. Senão, eu espero na calçada.

3. **Estruturas de Repetição (Loops): Automatizando Tarefas**
Às vezes, precisamos repetir uma ação várias vezes. Em vez de escrever o mesmo comando repetidamente, usamos laços como o while (enquanto) ou o for (para cada)

No cotidiano: Enquanto houver sopa no prato, continue comendo.
Em Python: Um contador que imprime números de 0 a 4 até que uma condição de parada seja atingida

4. **Funções: Organizando o Trabalho**
Uma função é um bloco de código que realiza uma ação específica e pode ser reutilizado em diferentes partes do programa
No cotidiano: Pense em um liquidificador. Você coloca as frutas (entrada), ele bate (processamento) e te entrega um suco (saída)
Em Python: Você define a função uma vez com def e a chama pelo nome sempre que precisar usar o "liquidificador" de código

5. **Indentação: A Organização Visual**
No Python, o recuo (espaços no início da linha) é obrigatório e serve para dizer ao computador quais instruções pertencem a qual bloco lógico
Se você não indentar corretamente, o programa apresentará um erro de sintaxe
